# 기능 문서 작성 가이드

## 위치

`docs/` 디렉토리에 기능별 마크다운 파일로 작성한다. 게임 규칙은 `docs/games/` 하위에 둔다.

## 작성 원칙

- **기술 용어를 사용하지 않는다**: 소켓 이벤트 키, Redis 키, 타입 필드명, 환경변수명, 파일 경로 등 구현 상세를 포함하지 않는다. 기능의 동작과 규칙만 서술한다.
- 구현 상세는 코드와 아키텍처 문서(`.claude/rules/architecture.md`)에서 관리한다.

### 예시

```
# 좋은 예
- 서버가 자동으로 랭킹을 등록한다
- 관리자가 로비 랭킹 카드에서 의심 기록을 삭제 가능

# 나쁜 예
- 서버가 `game-handler.ts`에서 자동으로 랭킹 등록
- `ranking:delete` 이벤트로 의심 기록을 삭제 가능
- Redis 키: `ranking:{gameType}:{difficulty}`
```

## 포맷 규칙

- 제목은 `#`, 섹션은 `##`, 하위 섹션은 `###`을 사용한다
- `#` 제목 바로 아래에 한 줄 요약을 작성한다
- 섹션(`##`) 사이는 빈 줄 하나로 구분한다
- 구조화된 데이터는 테이블로 작성한다
- 보충 설명은 테이블 아래 리스트(`-`)로 작성한다
- 리스트 항목에 굵은 키를 쓰지 않는다 (`- **키**: 값` X → `- 설명` O)
- 리스트 하위 항목은 들여쓰기 2칸 + `-`로 작성한다
- 섹션 내용이 한 줄이면 본문 텍스트로, 여러 항목이면 리스트로 작성한다
