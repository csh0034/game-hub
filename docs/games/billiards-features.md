# 3쿠션 당구 기능 상세

게임 규칙과 기본 진행은 `billiards.md`를 참고한다. 이 문서는 조작, 네트워크, 미구현 기능을 다룬다.

## 페이즈

```
aiming → simulating → scored / missed → aiming
```

- aiming: 조준 및 샷 입력 대기
- simulating: 서버 물리엔진 실행, 클라이언트에 프레임 브로드캐스트
- scored: 득점 판정, 같은 플레이어 턴 유지
- missed: 미득점, 다음 플레이어로 턴 전환

## 입력 처리

### 마우스

- 좌클릭 아래 드래그: 파워 조절 (10px 초과 시 진입, 200px로 최대 파워, 놓으면 샷 실행)
- 파워 5% 미만이면 샷 미실행
- 우클릭 좌우 드래그: 조준 방향 조정 (카메라가 함께 따라감)
- 우클릭 상하 드래그: 카메라 고도 조절 (위로 드래그 시 시점이 낮아짐)
- 휠: 줌인/아웃 (거리 1.2~6m)

### 키보드

| 키 | 동작 |
|------|------|
| 화살표 좌/우 | 조준 방향 조정 (1도 단위) |
| Shift + 화살표 좌/우 | 미세 조준 (0.2도 단위) |
| W/S | 당점 상하 조정 |
| A/D | 당점 좌우 조정 |
| Shift + WASD | 당점 미세 조정 |
| R | 당점 초기화 |

### 당점

- 최대 오프셋: 공 반지름의 70%
- 당점에 따라 탑스핀, 백스핀, 사이드스핀 발생
- 미스큐 위험도 표시 (85% 초과 시 경고)

### 샷 파라미터

- 방향(도), 파워(0~1), 당점 좌우 오프셋, 당점 상하 오프셋
- 새 턴 시작 시 가장 가까운 목적구 방향으로 자동 조준
- 드래그 → 속도 1.0~13.89 m/s 선형 매핑

## 가이드라인

- 조준 중 수구의 예상 경로를 점선으로 표시
- 미니 물리 시뮬레이션 기반 경로 예측 (스핀 커브, 쿠션 반사 반영)
- 목적구 히트 시 히트 지점까지만 표시

## 공 궤적 (트레일)

- 샷 시뮬레이션 중 수구(플레이볼)의 이동 경로만 잔상으로 표시
- 서버 프레임 수신 시 공 위치를 누적하여 경로를 기록한다
- 이전 위치와 0.001m 이상 차이가 날 때만 새 포인트를 추가하여 중복을 방지한다
- 최대 300개 포인트까지 기록
- 공 중심 높이에서 천 표면 바로 위(+0.001m)에 선으로 렌더링
- 트레일 두께: 4mm (5개 평행 라인으로 구현)
- 시뮬레이션 시작 시 궤적을 초기화한다
- 우측 상단 토글 스위치로 궤적 표시를 켜고 끌 수 있다 (기본값: ON)

## 네트워크

- 서버 권위 물리엔진: 서버에서 시뮬레이션 실행, 50ms 간격으로 프레임 브로드캐스트
- 프레임 데이터: 공 위치, 속도, 3축 스핀, 충돌 이벤트
- 클라이언트에서 프레임 간 보간(lerp) + 속도 기반 외삽으로 부드러운 렌더링

## 미구현 기능

| 기능 | 설명 |
|------|------|
| 사운드 효과 | 큐 타격, 공-공 충돌, 쿠션 충돌 등 |
| 리플레이 시스템 | 득점 샷 리플레이 재생, 프레임 슬라이더 |
| VAR (영상 판독) | 득점 유효성 이의 제기 및 투표 시스템 |
| 큐 앙각 조절 | 큐 들어올리기 각도 변경 (점프/마세이) |
| 도움말 오버레이 | 키보드 조작법 안내 화면 |
| AI 플레이어 | 컴퓨터 대전 |
