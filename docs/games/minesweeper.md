# 지뢰찾기 (Minesweeper)

## 개요

1인 퍼즐 게임. 보드에 숨겨진 지뢰를 피해 **모든 안전한 칸을 열면** 승리한다. 초급/중급/고급 3단계 난이도를 지원한다.

## 기본 정보

| 항목 | 내용 |
|------|------|
| 플레이어 수 | 1명 |

## 난이도

| 난이도 | 보드 크기 | 지뢰 수 | 안전한 칸 |
|--------|-----------|---------|-----------|
| 초급 (Beginner) | 9×9 (81칸) | 10개 | 71개 |
| 중급 (Intermediate) | 16×16 (256칸) | 40개 | 216개 |
| 고급 (Expert) | 16×30 (480칸) | 99개 | 381개 |

## 게임 진행

1. 첫 클릭 시 지뢰가 배치된다. **첫 클릭 위치와 주변 8칸은 안전**이 보장된다.
2. 칸을 열면(reveal) 인접한 8방향의 지뢰 수가 표시된다.
3. 인접 지뢰가 0인 칸을 열면 주변 칸이 자동으로 열린다 (**flood fill**).
4. 지뢰가 있다고 생각되는 칸에 깃발(flag)을 놓을 수 있다.
5. 깃발이 놓인 칸은 실수로 열리지 않도록 보호된다.
6. 불확실한 칸에는 물음표(?)를 표시할 수 있다.

## 조작

| 동작 | 설명 |
|------|------|
| 좌클릭 | 칸 열기 |
| 우클릭 | 순환: 깃발 → 물음표 → 원래 상태 |
| 양클릭 / 중클릭 | 주변 칸 일괄 열기 (숫자와 깃발 수 일치 시) |

## 물음표

우클릭을 반복하면 깃발 → 물음표 → 원래 상태 순으로 순환한다. 물음표는 깃발처럼 칸을 보호하지만 남은 지뢰 수에는 포함되지 않는다. 지뢰가 있을 수 있지만 확신이 없는 칸을 표시하는 용도로 사용한다.

## 양클릭 (Chord)

숫자가 표시된 칸에서 좌우 동시 클릭(또는 중클릭)을 하면, 주변에 놓인 깃발 수가 해당 숫자와 일치할 때 나머지 닫힌 칸을 한꺼번에 연다. 깃발이 잘못 놓여 있으면 지뢰가 열리며 패배한다.

- 양쪽 버튼을 누르고 있는 동안 열릴 대상 칸이 움푹 들어간 모양으로 표시된다
- 마우스를 움직이면 눌림 표시가 커서를 따라간다

## 승리 조건

지뢰가 아닌 **모든 안전한 칸을 열면** 승리한다.

## 패배 조건

**지뢰가 있는 칸을 열면** 패배한다. 패배 시 모든 지뢰 위치가 공개된다.

## 숫자 표시

열린 칸에 표시되는 숫자는 인접 8방향에 있는 지뢰의 개수를 나타낸다.

| 숫자 | 색상 |
|------|------|
| 1 | 파랑 |
| 2 | 초록 |
| 3 | 빨강 |
| 4 | 남색 |
| 5 | 갈색 |
| 6 | 청록 |
| 7 | 검정 |
| 8 | 회색 |

## Flood Fill

인접 지뢰 수가 0인 칸을 열면, BFS로 주변 8방향의 칸을 자동으로 연다. 이 과정은 인접 지뢰 수가 0이 아닌 칸(경계)을 만날 때까지 계속된다.

## 랭킹

난이도별 완료 시간 기준 Top 10 랭킹. 시간이 낮을수록 높은 순위.

- **타이머**: 서버 사이드 (`Date.now()`) — 첫 reveal 시 시작, 승리 시 종료
- **자동 등록**: 승리 시 서버가 자동으로 랭킹 등록 (클라이언트 제출 불가)

## 치팅 방지

| 조치 | 설명 |
|------|------|
| 최소 클리어 시간 | 초급 3초 / 중급 15초 / 고급 35초 미만은 랭킹 등록 거부 |
| 이동 속도 제한 | reveal 간 최소 50ms 간격 강제 (자동화 클릭 차단) |
| 관리자 삭제 | 관리자가 로비 랭킹 카드에서 의심 기록 삭제 가능 |
