# 요청사항 게시판

로비 내 탭으로 접근하는 기능 요청 게시판. 모든 접속자가 요청을 등록하고, 관리자가 상태를 관리한다.

## 라벨

| 라벨 | 설명 |
|------|------|
| feature | 기능 요청 (기본값) |
| bug | 버그 |
| improvement | 개선 |
| new-game | 게임 추가 |

- 등록 시 필수 선택
- 관리자가 모든 상태에서 변경 가능

## 상태

| 상태 | 설명 |
|------|------|
| open | 요청 (초기 상태) |
| in-progress | 진행중 |
| resolved | 완료 |
| rejected | 거부 |
| stopped | 중단 |

- 관리자가 드롭다운으로 상태를 즉시 변경
- 모든 상태에서 다른 상태로 자유롭게 전환 가능
- 이미 같은 상태인 경우 무시

## 노출 순서

상태별로 그룹화하여 표시하며, 각 그룹 내 정렬 기준은 다음과 같다.

| 상태 | 정렬 기준 | 방향 |
|------|-----------|------|
| 요청 | 등록 시간 | 최신순 |
| 진행중 | 진행 시작 시간 | 최신순 |
| 완료 | 완료 처리 시간 | 최신순 |
| 거부 | 거부 처리 시간 | 최신순 |
| 중단 | 중단 처리 시간 | 최신순 |

## 인라인 수정

관리자가 더블클릭하여 다음 필드를 즉시 수정 가능:

- 제목, 설명, 답변, 커밋 해시
- 답변이 없는 경우 "답변 추가" 버튼으로 새로 작성
- 완료 상태에서 커밋 해시가 없는 경우 "해시 추가" 버튼으로 새로 입력
- 빈 값으로 저장 시 해당 필드 삭제 (답변, 커밋 해시)

## 완료

- 관리자가 커밋 해시 입력 (선택)
- 입력 시 GitHub 커밋 페이지 링크 자동 생성
- 커밋 해시를 잘못 입력한 경우 더블클릭하여 바로 수정 가능

## 삭제

- 관리자가 요청사항 삭제 가능 (확인 다이얼로그 후 삭제)
